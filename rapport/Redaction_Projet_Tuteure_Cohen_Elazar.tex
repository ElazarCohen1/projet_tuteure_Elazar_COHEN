    \section{Introduction}
        Ce fichier présente mon projet tuteuré pour le deuxieme semestre de L3.
        Le projet est un bot discord dans un premier temps qui propose a un groupe de personne 
        des recettes personnalisé en fonction du prompt donnée par l'utilisateur.

    \section{Présentation du projet et fonctionnalités}

        \paragraph{Introduction}
            Le projet consiste à concevoir un bot Discord capable de proposer des recettes personnalisées aux utilisateurs d’un même groupe.
            Les utilisateurs pourront interagir avec le bot via des commandes textuelles.
            Contrairement à un simple système de recommandation de recettes, le bot créera des recettes personnalisées à partir des informations sur les utilisateurs :
            leurs goûts, les instruments de cuisine qu’ils ont à disposition ou encore leur budget.
            Le bot est conçu pour fonctionner dans un cadre collaboratif, ou pour être adapté à une seule personne.

        \paragraph{Objectifs du projet}
            L’objectif principal de ce projet est de développer un système capable de générer des recettes culinaires adaptées aux préférences et contraintes des utilisateurs.
            Le projet vise à montrer qu’il est possible d’utiliser un modèle de langage pour produire des contenus personnalisés de manière contrôlée, en tenant compte de données utilisateurs structurées.
            Un autre objectif est de proposer un outil simple d’utilisation, intégré à une plateforme de communication existante, afin de faciliter l’adoption par les utilisateurs.

    \section{Analyse des besoins}

        L’utilisation du bot répond à un besoin courant : proposer des idées de repas adaptées à des personnes ayant des goûts et des contraintes différentes.
        Dans un contexte de groupe, il est souvent difficile de trouver une recette qui convienne à tout le monde.
        Le bot vise à simplifier cette prise de décision en générant automatiquement des propositions cohérentes avec les préférences exprimées.

        Les besoins principaux identifiés sont :
        \begin{itemize}
            \item une personnalisation des recettes 
            \item une prise en compte de plusieurs utilisateurs 
            \item une interaction simple et rapide 
            \item une génération de recettes originales et adaptées.
        \end{itemize}
 
    \section{Choix techniques}

        \paragraph{Langage et environnement}
            Le projet sera développé en Python.
            Ce langage a été choisi pour sa simplicité, sa lisibilité et son écosystème riche, notamment dans le domaine de l’intelligence artificielle.
            Python permet également une intégration simple avec des bibliothèques dédiées à la création de bots Discord.
        \paragraph{Plateforme Discord}
            Discord a été choisi comme plateforme de déploiement car il s’agit d’un outil largement utilisé pour la communication de groupe.
            L’intégration du bot dans un serveur Discord permet une interaction naturelle avec plusieurs utilisateurs dans un même espace.

        \paragraph{Modèle de langage}
            Le bot s’appuiera sur un modèle de langage exécuté localement.
            Ce choix permet de garder un contrôle sur le fonctionnement du système et d’éviter la dépendance à des services externes.
            Le modèle sera utilisé pour générer des recettes à partir d’un contexte fourni par le système.
     
    \section{Conception générale du système}

        Le fonctionnement du bot repose sur plusieurs composants :
        \begin{itemize}
            \item une interface utilisateur via Discord 
            \item un module de gestion des profils utilisateurs 
            \item un module de génération de recettes 
            \item un modèle de langage chargé de produire les recettes.
        \end{itemize}

        Lorsqu’un utilisateur effectue une demande, le bot récupère les informations nécessaires, construit un contexte adapté et transmet ces informations au modèle de langage afin de générer une recette personnalisée.

    \section{Travail fourni et limites du projet}

    \subsection{Travail fourni}

        \paragraph{Fonctionnement général}
            Le bot permet à un utilisateur de demander une recette personnalisée directement depuis Discord à l’aide d’un prompt textuel.
            Une fois la demande envoyée, le système traite les informations et génère une recette adaptée au contexte fourni par l’utilisateur.
            Lorsque le modèle de langage est exécuté localement, le temps de réponse moyen est d’environ deux minutes.

            Le projet intègre plusieurs composants fonctionnant ensemble :
            \begin{itemize}
                \item un bot Discord permettant l’interaction avec les utilisateurs ;
                \item une base de données relationnelle contenant les recettes ;
                \item un système de recherche vectorielle basé sur les embeddings ;
                \item un modèle de langage local chargé de générer les recettes.
            \end{itemize}

        \paragraph{Fonctionnement interne}
            Lorsqu’un utilisateur effectue une demande, le texte du prompt est transformé en embedding vectoriel.
            Le système effectue ensuite une recherche par similarité cosinus dans la table contenant les embeddings des recettes afin de récupérer les recettes les plus pertinentes par rapport à la demande.

            Les recettes les plus proches sont ensuite transmises au modèle de langage Mistral exécuté localement via Ollama.
            Le modèle utilise ces recettes comme contexte afin de générer une nouvelle recette cohérente et adaptée à la demande de l’utilisateur.

            Une fois la génération terminée, le bot envoie automatiquement la réponse dans Discord et notifie l’utilisateur que sa recette est prête.

        \paragraph{Travail réalisé sur les données}
            Une partie importante du projet a concerné le traitement et la normalisation des données.
            Les recettes récupérées depuis différentes sources possédaient des formats variés et souvent incohérents.
            Il a donc été nécessaire de nettoyer les données, standardiser les ingrédients et restructurer les informations afin de pouvoir les exploiter correctement dans la base de données.

            L’utilisation du parser \texttt{ingredient\_parser} a permis d’améliorer significativement la qualité de la normalisation des ingrédients et de rendre les données plus homogènes.

        \paragraph{Optimisation des performances}
            Une partie importante du travail a porté sur l’optimisation des performances d’insertion des données dans la base de données.
            Au début du projet, les recettes étaient insérées ligne par ligne directement depuis le programme Python.
            Cette méthode fonctionnait correctement pour de petites quantités de données, mais elle est rapidement devenue très lente lorsque le nombre de recettes a augmenté.

            Chaque insertion provoquait une requête SQL ainsi qu’une transaction indépendante, ce qui entraînait un coût important en temps d’exécution.
            Avec cette méthode, l’insertion complète d’environ 10\,000 recettes pouvait prendre plus de deux heures et demie.

            Afin d’améliorer les performances, une première optimisation a consisté à regrouper les insertions par lots.
            Les recettes étaient alors insérées par batchs d’environ 5\,000 lignes afin de réduire le nombre de requêtes et de transactions effectuées.
            Cette approche a permis une amélioration des performances, mais le temps d’insertion restait encore trop élevé pour une utilisation efficace.

            Après plusieurs recherches et discussions avec Monsieur Forax, il est apparu qu’une meilleure solution consistait à utiliser des fichiers CSV afin d’effectuer des insertions massives directement dans PostgreSQL.
            Le système a donc été entièrement modifié.

            Désormais, le programme génère un fichier CSV complet pour chaque table de la base de données.
            Toutes les lignes sont préparées à l’avance, puis insérées en une seule opération grâce aux outils d’importation de PostgreSQL.
            Cette méthode réduit fortement le nombre d’échanges entre Python et la base de données.

            Afin de conserver correctement les relations entre les tables, les identifiants sont également générés manuellement lors de la création des fichiers CSV.
            Cela permet de relier correctement les recettes, les ingrédients et les embeddings sans avoir à effectuer des requêtes supplémentaires pour récupérer les clés générées automatiquement.

            Grâce à cette nouvelle méthode, le temps d’insertion d’environ 10\,000 recettes est passé de plus de deux heures et demie à environ deux minutes.

            Cependant, il est difficile de réduire davantage ce temps de traitement.
            En effet, la majorité du temps restant est désormais consacrée à la génération des embeddings vectoriels.
            Cette étape représente environ 90\,\% du temps total d’insertion des données.

            Les embeddings étant générés localement sur le processeur (CPU) et non sur une carte graphique (GPU), les calculs restent relativement coûteux en temps.
            Les performances pourraient être améliorées avec un GPU dédié ou une infrastructure plus puissante, mais cela dépassait la capacité de mon ordinateur.
    \subsection{Limites du projet}

        \paragraph{Fonctionnalités non terminées}
            L’objectif initial du projet était de proposer une personnalisation avancée des recettes en fonction des profils utilisateurs.
            Le bot devait notamment enregistrer les préférences, allergies, budgets ou équipements de cuisine des utilisateurs afin de générer des recettes adaptées à chaque personne ou à un groupe complet.

            Cependant, le temps nécessaire à la mise en place du système de génération et du traitement des données a été plus important que prévu.
            Certaines fonctionnalités avancées n’ont donc pas pu être entièrement développées avant la fin du projet.

        \paragraph{Limites techniques}
            Plusieurs difficultés techniques ont été rencontrées durant le développement :
            \begin{itemize}
                \item la compréhension et la mise en place du fonctionnement du RAG ;
                \item la normalisation automatique des données issues de différentes sources ;
                \item la cohérence des recettes générées par le modèle de langage ;
                \item les performances de traitement et d’insertion dans la base de données ;
                \item les limitations matérielles liées à l’exécution locale d’un modèle de langage.
            \end{itemize}

            Le fonctionnement local du modèle permet de conserver une certaine indépendance vis-à-vis de services externes, mais cela implique également des temps de génération plus élevés et des performances limitées par le matériel disponible.

        \paragraph{Axes d’amélioration}
            Plusieurs améliorations pourraient être apportées au projet dans le futur :
            \begin{itemize}
                \item ajout d’un véritable système de profils utilisateurs ;
                \item prise en compte des allergies et restrictions alimentaires ;
                \item apprentissage à partir des retours utilisateurs ;
                \item amélioration de la qualité des recettes générées ;
                \item optimisation supplémentaire des performances ;
                \item déploiement sur un serveur dédié afin de réduire les temps de réponse.
            \end{itemize}         
    \section{Conclusion}

        Ce projet tuteuré a pour objectif de mettre en œuvre un bot Discord capable de générer des recettes personnalisées à l’aide d’un modèle de langage.
        Il permet d’aborder des notions telles que la personnalisation, la gestion de données utilisateurs et l’utilisation raisonnée de l’intelligence artificielle.
        Le projet constitue une application concrète des compétences acquises durant la licence informatique.
